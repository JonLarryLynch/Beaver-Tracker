🦫 Import historical sightings

This is a one-time bulk-loading tool, not part of the public site. It reads a CSV export of real wildlife observations (from iNaturalist, GBIF, or eBird) and adds them to your Firestore sightings collection as historical records, so the map and Patterns tab have a real starting dataset instead of an empty one.

1. Get a CSV export

The easiest source: go to inaturalist.org/observations, use the location filter to search "Lincoln Park, Chicago" (or draw a custom box around the park), optionally filter by species if you only want certain animals, then use the Filters → Download option to request a CSV export. iNaturalist emails you a download link, usually within a few minutes. GBIF.org and eBird's "Explore Data" tool work similarly if you want a broader or bird-specific dataset instead.

2. Upload the CSV here

3. Import

Each imported row is tagged source: "historical-import" so you can tell it apart from real community sightings later if you ever want to. Running this twice on the same file won't create duplicates, it uses each observation's original ID.

A note on attribution: most iNaturalist and GBIF observations are shared under Creative Commons licenses that ask for credit to the original observer. This tool keeps a link back to the original observation in each imported sighting's notes for that reason. Worth a quick read of the license column in your export before publishing anything drawn from someone else's photo.
